Deep Research: Agent-User Interaction & Feedback

Last Updated: May 2025 | Focus: User Feedback Loops, Notification Design, Interaction Patterns

1. Overview

Agent-User Interaction focuses on the bidirectional communication between AI agents and human users: how agents notify users, how users provide feedback, and how this feedback shapes agent behavior. Unlike general human-AI interaction, this research specifically addresses interaction mechanics—the design of feedback channels, notification timing, and correction interfaces that enable effective human-agent collaboration.

The core insight from recent research: feedback is the primary mechanism for agent improvement. Without structured user feedback, agents cannot calibrate behavior, correct errors, or adapt to user preferences. Yet most agent systems treat feedback as an afterthought—simple "approve/reject" buttons rather than rich, actionable correction channels.

Key Finding: IJCAI 2025 Survey identifies four feedback types: explicit (direct corrections), implicit (behavior signals), strategic (goal refinement), and operational (execution guidance). Each requires different interface design and integration mechanisms.

2. User Feedback Mechanisms

2.1 Feedback Type Taxonomy

💬

Explicit Feedback

Direct user corrections, approvals, rejections. Most common but highest user effort. Examples: "Change this to X", "No, that's wrong", "Good job".

Text correction Approval button Rejection flag
👁️

Implicit Feedback

Behavior signals without active user input. Time spent, scroll patterns, acceptance rate, edit frequency. Requires instrumentation to capture.

Time on task Accept rate Edit frequency
🎯

Strategic Feedback

Goal refinement, priority adjustment. Shapes agent's understanding of objectives. Higher-level than operational corrections.

"Focus on speed" "Be more concise" "Prioritize accuracy"
⚙️

Operational Feedback

Step-by-step intervention during execution. Real-time guidance on actions. HAC (Human-Agent Control) policy determines intervention timing.

Pause agent Redirect action Modify parameters

2.2 Feedback Integration Pipeline

How user feedback becomes agent improvement:

Step 1 Capture: Interface collects user input (explicit) or behavior data (implicit)
Step 2 Interpret: Agent parses feedback intent—correction type, scope, severity
Step 3 Apply: Immediate adjustment to current task or stored for future reference
Step 4 Learn: Feedback feeds into agent memory, fine-tuning, or policy adjustment

2.3 Feedback Interface Design

  • Natural language input: Users describe corrections in plain language, not menus/buttons. More expressive but harder to parse.
  • Inline editing: Users directly edit agent outputs. Immediate, visual, but limited to text/content.
  • Structured feedback forms: Dropdowns, sliders, checkboxes. Easier to process but constrains expression.
  • Rejection + explanation: "Why was this wrong?" prompts after rejection. Enables root cause analysis.
  • Rating + correction: Combine satisfaction rating (1-5) with specific fix request.

3. Agent Notification & Proactivity

3.1 When Should Agents Notify Users?

Agents must balance helpfulness against interruption cost. Key triggers for proactive notification:

Trigger Type When Agent Notifies Example
Completion Task finished, result ready "Your report is complete"
Blocker Cannot proceed without input "I need access to file X"
Uncertainty Confidence below threshold "Not sure which option you prefer"
High-Stakes Irreversible action pending "Ready to delete 50 files"
Error Action failed, needs guidance "API call failed, should I retry?"
Discovery Found relevant information "I noticed this might interest you"

3.2 Notification Timing Design

HCI research on interruption timing informs agent notification design:

  • Immediate notification: Urgent blockers, errors requiring decision. Interrupts user flow but prevents bigger failures.
  • Deferred notification: Non-urgent completions, discoveries. Batch and deliver at natural breakpoints.
  • Threshold-based: Confidence/risk thresholds determine immediacy. High risk → immediate; low risk → deferred.
  • User-controlled: Preference settings for notification frequency, channels (email, push, in-app).

3.3 Clarification Questions

When agents encounter ambiguity, clarification questions resolve uncertainty:

Open-ended

"What would you like to know more about?" — Flexible but requires more user effort.

Closed Choice

"Did you mean X or Y?" — Efficient, structured, but constrains options.

Confirmation

"Just to confirm, you want..." — Verifies understanding before acting.

Slot-filling

"What date for this meeting?" — Gathers specific required parameters.

4. Interaction Patterns

4.1 Mixed-Initiative Interaction

Mixed-initiative systems allow both user and agent to take control at different times. Key dimensions: autonomous vs interactive, user-led vs agent-led. The system offers suggestions, warnings, clarifications while allowing user intervention.

  • Agent initiative: Proactive suggestions, auto-completions, warnings
  • User initiative: Direct commands, corrections, overrides
  • Turn-taking: Structured alternation between agent action and user input

4.2 Approval Workflows

Three architectural patterns for user approval:

Synchronous Approval

Use: Irreversible, high-stakes decisions

Agent pauses execution pending user authorization. Zero risk of unapproved action but introduces latency.

Examples: File deletion, financial transactions, API calls to external systems

Asynchronous Audit

Use: Reversible, lower-risk decisions

Agent executes immediately, logs for later review. Near-zero latency but delayed error detection.

Examples: Content drafts, internal updates, non-critical operations

Confidence-Based Routing

Use: Variable risk scenarios

High confidence → autonomous execution. Low confidence → human review. Matches oversight intensity to actual risk.

Threshold tuning based on production task distributions, not generic benchmarks

4.3 Error Recovery Interaction

When users reject agent outputs, recovery strategies maintain trust:

  • Immediate retry: Agent offers alternative approach
  • Explanation request: "What specifically was wrong?" to understand rejection reason
  • De-escalation: Offer simpler, more conservative action
  • Human takeover: User completes remaining steps manually
  • Undo + revert: Roll back changes, restore previous state

5. User Input Processing

5.1 Input Channels

Text Input

Natural language commands, corrections, clarifications. Primary channel for most agents. Requires NLU parsing.

Most expressive

UI Controls

Buttons, sliders, dropdowns, checkboxes. Structured input easier to process. Common for approval/rejection.

Most precise

Direct Editing

Users edit agent outputs inline (documents, code). Visual, immediate feedback. Implies acceptance + correction.

Most visual

Voice Input

Spoken commands, especially for hands-free contexts. Requires speech recognition + intent parsing.

Hands-free

Behavior Signals

Click patterns, time spent, scroll depth. Implicit feedback from user actions, not explicit input.

Passive collection

File/Data Upload

Documents, images, datasets provided to agent. Rich context input beyond text descriptions.

Context-rich

5.2 Intent Interpretation

Agent must parse user input to extract actionable intent:

  • Command detection: Is this a new task, correction, or clarification?
  • Scope parsing: Does feedback apply to current output, general behavior, or specific setting?
  • Severity assessment: Major error requiring full revision vs minor tweak?
  • Preference inference: What implicit preference does this correction reveal?

5.3 Context Carry-Forward

User input must persist across sessions for agent learning:

  • Session memory: Feedback applies to current conversation only
  • User profile: Preferences stored per-user, applied across sessions
  • Task memory: Corrections remembered for similar future tasks
  • Global learning: Feedback improves base model (fine-tuning, RL)

6. Feedback Effect on Agent Behavior

6.1 Immediate Application

How feedback shapes current task execution:

  • Output revision: Agent modifies result based on correction
  • Plan adjustment: Changes future steps in multi-step workflow
  • Tool reselection: Switches to different approach/tool
  • Confidence update: Adjusts self-assessment based on user signal

6.2 Long-Term Learning

Feedback accumulation improves agent over time:

  • Preference learning: User corrections reveal stable preferences (concise output, formal tone)
  • Error pattern recognition: Repeated corrections identify systematic weaknesses
  • Behavioral calibration: Feedback tunes agent's decision thresholds
  • Fine-tuning: Corrections feed into model training pipelines

6.3 Self-Correction Mechanisms

Research frameworks for agent self-improvement from feedback:

Framework Mechanism Feedback Type
Self-Refine Iterative refinement with self-feedback Internal (self-generated)
CRITIC Tool-interactive critiquing External (tool verification)
Reflexion Verbal reinforcement with memory Internal + External

7. Interaction Design Principles

Transparency

Users understand what agent is doing, why, and with what confidence. Progress indicators, execution traces, reasoning visibility.

Controllability

Users can pause, redirect, override agent at any point. Granular control over autonomy level. No forced autonomous execution.

Expressiveness

Feedback channels allow rich input, not just approve/reject. Natural language corrections, inline edits, structured explanations.

Feedback Efficiency

Minimize user effort for corrections. Inline editing easier than text description. Progressive disclosure for complex feedback.

Trust Calibration

Make agent limitations visible. Confidence scores, uncertainty disclosure, failure mode transparency. Overtrust prevention.

Recovery Grace

Failed actions have clear undo paths. Users can revert without blame. Error explanation helps understanding.

7.1 Feedback Fatigue Prevention

  • Batch approvals: Group similar decisions, ask once instead of per-item
  • Threshold tuning: Only ask for input when confidence truly low; avoid unnecessary interruptions
  • Smart defaults: Learn preferences, reduce repeated questions
  • Defer non-urgent: Accumulate minor questions, deliver at natural breakpoints

7.2 Mental Model Alignment

  • Capability disclosure: What the agent can/cannot do
  • Limitation visibility: Known failure modes, edge cases
  • Decision reasoning: Why agent chose this approach
  • Confidence interpretation: What 85% confidence actually means

9. Challenges & Open Problems

  • Feedback interpretation: Natural language corrections ambiguous—"make it better" lacks actionable specificity
  • Implicit feedback noise: Behavior signals (scroll, time) noisy; difficult to attribute intent
  • Feedback latency: User corrections arrive after completion; harder to apply mid-execution
  • Preference stability: User preferences change over time; stored feedback may become stale
  • Cross-user learning: Feedback from one user shouldn't negatively affect another
  • Feedback storage: Where/how to persist feedback—session, user profile, task memory?
  • Notification timing: Optimal interruption timing varies by user, task, context
  • Trust erosion: Repeated errors + corrections reduce user willingness to provide feedback

10. Future Directions

  • Interaction models: Native interactivity in LLMs, not bolted-on notification systems (Thinking Machines Lab)
  • Real-time feedback: Streaming interfaces where users interject mid-execution, not after completion
  • Multimodal feedback: Voice corrections, gesture input, visual annotations beyond text
  • Preference learning: Automatic extraction of stable preferences from correction history
  • Clarification generation: Better agent questions when uncertain—contextual, specific, not generic
  • Feedback economics: Measuring user effort cost vs improvement gain; optimal feedback frequency
  • Cross-session memory: Feedback persistence architecture—what to store, how to apply
  • Trust recovery: After errors, how agents rebuild user willingness to engage
