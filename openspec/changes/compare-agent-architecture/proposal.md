## Why

对比分析当前 ollama-mlflow-demo 代码仓库与 Hermes-Agent 的 Agent 架构，了解两者在设计理念、核心功能、代码规模上的差异，为后续架构演进和功能借鉴提供参考依据。

当前仓库是一个轻量级的 Agent Gateway 实现，而 Hermes-Agent 是一个成熟的生产级 AI Agent 框架。通过对比可以发现我们架构的优势和不足，明确未来发展方向。

## What Changes

- 创建架构对比文档，分析两个仓库的核心差异
- 对比以下维度：
  - **代码规模与结构**: 文件数量、核心代码行数、模块划分
  - **核心 Agent 实现**: 对话循环、工具执行、并发处理
  - **Session 管理**: 存储方式、压缩策略、搜索能力
  - **工具系统**: 注册机制、并发安全、MCP 支持
  - **Skills 系统**: 加载机制、技能库规模
  - **Gateway/消息平台**: 支持的平台数量、适配器设计
  - **权限与安全**: 权限控制、审批机制
  - **记忆系统**: 持久化、用户建模
